# プレイスキル発動の修正リスト

## 背景

ルール上、「プレイする」と記載されたスキルでカードがステージに出された場合、
そのカードのプレイスキルが発動すべき。楽屋にプレイ（ゲスト）の場合は発動しない。

現状、`ZoneOps.play_to_stage_from_zone` でカードを配置しているが、
配置されたカードのプレイスキルを発動する処理が欠落している。

## 「プレイ」ではないカード（修正不要）

以下は意図的に「移動する」「入れ替える」等の表記を使用しており、プレイスキルは発動しない。

- 013 常闇トワ: 「対象にできない」（保護効果のみ）
- 027 角巻わため: 「ステージにプレイする事ができない」（制限効果のみ）
- 032 癒月ちょこ: 「オープンする。その際、プレイ時能力が発動しない」（明示的に不発動）
- 045 セレス・ファウナ: 「ステージに移動する」
- 047 ハコス・ベールズ: 「入れ替える」

## 実装方針

スキル解決中に別カードのプレイスキルを発動する仕組みが必要。
現在 `sub_skill_card_id` パターン（034等）はスキルコード内で直接 `execute_skill` を呼んでいるが、
これはカードを移動せずスキルだけ発動するケース向け。

「プレイする」パターンでは：
1. カードをステージに配置（ZoneOps）
2. 配置したカードのプレイスキルを発動（GameController._try_trigger_play_skill 相当）

→ SkillResult に「プレイスキル発動を要求する」フラグまたは情報を付与し、
  GameController のスキル解決ループで後続スキルとしてスタックに積む方式が望ましい。
